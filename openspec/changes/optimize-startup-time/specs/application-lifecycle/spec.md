## ADDED Requirements

### Requirement: 三秒内进入可交互主页面

系统 SHALL 将启动完成定义为从 LogCrate 进程入口开始，到 React 主页面完成首个绘制帧且主导航与监控目录区域能够响应用户输入。原生启动画面、WebView 页面加载完成事件或进度条到达 100% MUST NOT 单独视为启动完成。在参考设备的发布构建常规启动基准中，连续十次样本至少九次 MUST 在 3 秒内达到该里程碑，任何一次 MUST NOT 超过 5 秒。

#### Scenario: 常规启动进入主页面

- **WHEN** 用户在已安装受支持 WebView 运行时的参考设备上启动发布版 LogCrate
- **THEN** 原生反馈立即可见，随后主页面约三秒内完成首帧并可接收导航和监控目录区域的输入

#### Scenario: 后台恢复尚未完成

- **WHEN** 主页面已完成首个可交互帧但监控 watcher、搜索、更新检查或缓存维护仍在执行
- **THEN** 用户可以操作主导航和已就绪的核心界面，未完成的数据使用稳定占位或随后增量更新

#### Scenario: 启动性能发生退化

- **WHEN** 发布版十次常规启动中超过一次未在 3 秒内可交互，或任一样本超过 5 秒
- **THEN** 启动性能验收失败且变更不得标记完成

### Requirement: 启动关键路径与后台任务隔离

系统 SHALL 在原生首帧之后优先创建并加载主 WebView。主页面显示 MUST NOT 等待监控目录 watcher 建立、搜索数据库或索引初始化、自动更新检查、陈旧缓存清理及其他非首屏任务。后台任务 MUST 在首屏关键工作完成调度后执行，并 MUST NOT 通过同步磁盘访问、锁等待或持续资源竞争阻塞 UI 线程和 WebView 首帧。

#### Scenario: 搜索功能开启

- **WHEN** 用户已启用文件搜索并启动 LogCrate
- **THEN** 主页面先达到可交互状态，搜索后端随后在后台初始化且不延迟首屏

#### Scenario: 恢复多个监控目录

- **WHEN** 持久化配置包含多个监控目录或目录所在磁盘响应较慢
- **THEN** 主页面不等待所有 watcher 建立，目录状态就绪后增量显示或更新

#### Scenario: 启动动画与 WebView 并行

- **WHEN** 原生进度画面正在显示且 WebView 运行时正在初始化
- **THEN** 原生绘制保持可见和流畅，但不会以高频满窗口重绘显著争用 WebView 初始化资源

### Requirement: 可诊断的启动阶段计时

系统 SHALL 使用单调时钟记录进程入口、原生首帧、WebView 创建请求、WebView 创建完成、页面导航完成、React 挂载、主页面可交互帧、核心状态就绪及后台模块调度等启动阶段。阶段数据 MUST 保留在本地、不得上传远程服务，并 MUST NOT 通过 UI 线程同步磁盘写入延长启动时间。

#### Scenario: 生成发布版启动基准

- **WHEN** 开发者运行启动性能基准
- **THEN** 工具输出每次启动的总耗时和阶段明细，以及十次样本的 P50、P90 与最大值

#### Scenario: 启动超过性能上限

- **WHEN** 某次启动超过 5 秒才达到可交互里程碑
- **THEN** 本地阶段记录能够指出主要耗时区间，而不是只显示模拟进度百分比

#### Scenario: 正常用户启动

- **WHEN** 用户正常启动 LogCrate 且未运行性能基准
- **THEN** 启动计时不进行网络上传，也不在 UI 线程同步写入日志文件
