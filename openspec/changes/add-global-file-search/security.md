# Windows 索引服务威胁模型

## 资产与信任边界

- 受保护资产：本机文件名与目录结构、NTFS 卷句柄、USN Journal 状态以及 Windows 服务控制权限。
- 普通用户边界：LogCrate GUI 始终以当前交互用户运行，只接收文件名元数据，并在用户打开可见结果时自行校验路径。
- 高权限边界：`LogCrateIndex` 服务仅持有枚举本机 NTFS MFT/USN 所需权限；安装、修复和卸载仍由 Windows 的管理员批准流程控制。
- IPC 边界：仅接受本机 named pipe 客户端，拒绝远程客户端；ACL 只允许 SYSTEM、Administrators 和 Interactive Users。

## 允许的命令面

版本 2 IPC 只允许以下固定消息，卷参数必须是单个 ASCII 盘符：

1. 协议握手；
2. 枚举指定 NTFS 卷的 MFT 文件名记录；
3. 查询指定卷的 USN Journal 标识与范围；
4. 在给定 journal ID 和有界 USN 范围内读取增量记录。

协议不接受任意路径、程序、命令行或文件句柄。服务不提供写文件、删除、重命名、打开内容、网络访问或通用提权接口。

## 防护措施

- named pipe 使用受保护 DACL 和 `PIPE_REJECT_REMOTE_CLIENTS`。
- 单帧最大 8 MiB、单批最大 131,072 条记录，同时最多处理 4 个客户端。
- 未知协议版本、未知消息、越界字段、超大帧、尾随数据和非 UTF-8 文本全部拒绝。
- 客户端断开会令流式批次写入失败，该错误向上传播并终止当前 FSCTL 枚举，作为取消语义。
- GUI 在打开文件、加入监控或显示目录前重新检查路径，已删除或移动的索引项会从持久化索引移除。
- 服务二进制不链接 HTTP 客户端，也没有网络监听代码；文件内容读取和预览只发生在普通用户 GUI 中。

## 残余风险与发布要求

- 具有交互登录权限的本机用户可以通过 pipe 请求文件名元数据；这是全盘搜索功能的必要能力，安装界面必须明确说明文件名隐私影响。
- 文件名和目录结构可能包含敏感信息，索引只存于本机应用数据目录，不上传；卸载/清理流程必须删除本地索引。
- 发布包必须对 GUI 与服务二进制签名，并验证服务 DACL、按需启动配置、升级替换和卸载清理。
- 对 IPC 和 MFT/USN 解析器持续运行固定夹具、边界测试和随机输入属性测试；任何解析失败都必须局部失败，不能导致服务执行未定义操作。
