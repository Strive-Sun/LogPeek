# Change: 建立 zip 日志免解压监控最小闭环 (M1)

## Why
LogPeek 的核心价值是"下载日志压缩包 → 免解压 → 直接查看"。M1 需要打通这条链路的最小可用版本,验证产品的核心假设:用户往监控目录丢一个 zip,软件自动识别、顶栏提示、免解压查看,全程无需手动解压。范围收敛到 zip 与裸文本文件,但查阅能力目标为解压后 ≤ 2GB 单文件流畅查看(目标场景压缩包可达几百 MB)——该目标无法用"全量读入内存"支撑,因此 M1 直接纳入行索引 + 窗口化加载能力(原属 M2)。为让几百 MB 压缩条目打开后随机跳转/滚动无感,M1 采用方案 A:打开时一趟顺序解压到内部临时缓存文件(代价是占等量磁盘,用完即清);免磁盘放大的 Deflate 重启点索引留待 M2。多格式(tar.gz/7z/rar)与搜索/过滤等仍留到后续里程碑。

## What Changes
- 新增**多目录监控 + 持久化**:配置多条被监控目录并同时监控,列表持久化到本地配置、重启后自动恢复;对新到达文件做到达完整性检测(避免处理下载中的半成品)
- 新增**裸文本日志文件支持**:监控目录中的 `.log`/`.txt` 等裸文件与压缩包一同被识别、通知与查看(passthrough reader,无需解压)
- 新增**可配置后缀筛选**:用户自定义显示的文件后缀,过滤目录树与通知,规则随目录列表持久化
- 新增**zip 归档免解压读取 + 目录树惰性展开**:列出包内条目(仅读中央目录清单、零解压零落盘)、按需流式打开单个条目,内容不落地磁盘;目录树把压缩包"虚构展开"为内部文件,展开时才读清单
- 新增**顶栏新日志提示**:新日志包到达时在顶栏提示并计数;查看一个新包计数减一,另提供"全部标记已读"一键清零
- 新增**日志查看(行索引 + 窗口化加载)**:后台流式建行偏移索引(支持边建边读),前端虚拟滚动按需请求可视区行范围,支持解压后 ≤ 2GB 单文件流畅查看;压缩条目打开时一趟解压到临时缓存以支持随机访问无感(方案 A);含打开/跳转/滚动无感体验、文本编码检测与解码(UTF-8/GBK/UTF-16 + BOM)、LF/CRLF 兼容、超长行截断、查看会话生命周期管理(关闭释放 + LRU 回收 + 缓存清理)
- 新增**归档读取安全边界**:以实际读取字节数熔断大小上限(防 zip bomb),加密/分卷归档返回明确错误

## Impact
- Affected specs: `directory-monitoring`、`archive-reading`、`log-notification`、`log-viewing`(均为新增能力)
- Affected code:
  - `src-tauri/src/watcher.rs`(目录监控)
  - `src-tauri/src/archive/`(归档读取,含 `ArchiveReader` trait、zip 实现与裸文本 passthrough 实现)
  - `src-tauri/src/index.rs`(行偏移索引 / 窗口化加载 / 编码解码 / 会话生命周期）
  - `src-tauri/src/main.rs`(命令/事件注册)
  - 前端:目录树组件(压缩包惰性展开)、后缀筛选控件、条目列表组件、顶栏提示组件、虚拟滚动文本视图组件
- 范围外(本 change 不做):tar.gz/7z/rar 多格式、搜索/过滤/高亮、Deflate 重启点索引与采样索引优化(M2)、解压后 >2GB 超大文件
