# Change: 持久化日志工作区并提示外部文件变化

## Why

当前退出应用会丢失所有已打开选项卡，重新启动后用户必须再次定位并打开日志。应用运行期间，目录 watcher 虽能发现文件修改和删除，但删除事件会直接关闭相关选项卡，修改事件也不会询问是否重新加载，容易丢失排查上下文或继续查看过期内容而不自知。

## What Changes

- 持久化已打开选项卡、可见/折叠顺序和活动选项卡，启动时恢复工作区。
- 恢复时先验证磁盘源文件；已不存在的条目不恢复，并明确提示源文件已删除。
- 恢复的非活动选项卡保持休眠，用户激活时再建立读取会话，避免启动时同时索引全部日志。
- 已打开裸日志被外部修改时询问是否重新加载；忽略后继续阅读当前缓存快照。
- 已打开归档文件被外部修改时，对该归档及其嵌套条目共用一次提示；确认后重新加载相关活动条目，其余条目转为休眠并在激活时读取新版本。
- 已打开文件或归档被外部删除时提示“文件已被删除”；若受影响条目存在会话缓存，则优先提供当前活动条目的只读快照，否则提供该源下第一个可用缓存的“另存为”操作。现有缓存保留到用户关闭选项卡或退出应用，但已删除源不再写入下次恢复状态。
- 应用自身执行的删除操作沿用已有确认流程，不重复显示“外部删除”提示。

## Impact

- Affected specs: `log-workspace`, `directory-monitoring`
- Affected code: `src/App.tsx`, `src/util/workspace.ts`, `src/components/ConfirmDialog.tsx`, API 文件状态查询与中英文文案
- 不改变只读定位：不会写回已修改或已删除的源路径；用户只能通过系统保存窗口把当前缓存另存为新文件。
