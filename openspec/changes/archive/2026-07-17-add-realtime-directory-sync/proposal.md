# Change: 新增监控目录树实时同步

## Why

当前目录树只在启动、应用内文件操作和“新日志到达”通知后全量刷新。外部删除、重命名以及未被日志分类器识别的文件变化不会触发刷新，导致界面长期显示过期内容；如果简单地在每个文件事件后重扫所有目录，又会在批量复制时造成重复 IO 和事件积压。

## What Changes

- 将目录结构同步与新日志稳定检测/通知解耦，创建、删除、重命名和修改事件均可更新目录树。
- 使用系统原生 `notify` 事件进行增量同步，按监控目录在短时间窗口内合并事件，避免事件风暴重复刷新。
- 后端发送受影响目录的结构变化批次，前端只更新对应目录和节点，不再因单个事件重扫所有监控目录。
- 目录库存包含当前已加载目录的全部直接子项（普通文件与文件夹）；后缀筛选只决定文件展示和通知，不隐藏文件夹。
- 子目录采用懒加载递归浏览：展开时读取并监听该目录，折叠或节点移除时释放对应监听，避免启动时遍历整个目录树。
- 文件稳定检测移至独立后台任务，慢速写入不阻塞删除、重命名等结构事件。
- 对不完整、模糊或溢出的文件系统事件执行受影响目录的兜底重扫，恢复最终一致性。
- 增加事件合并、创建/删除/重命名、过滤切换、慢速写入和大批量事件测试。

## Impact

- Affected specs: `directory-monitoring`
- Affected code: `src-tauri/src/watcher.rs`、`src-tauri/src/lib.rs`、`src/api/*`、`src/App.tsx`、目录树状态更新工具及测试
- Performance: 空闲时继续使用系统 watcher、无定时轮询；变化时成本与受影响节点/目录相关，不与“全部目录 × 每个事件”相乘
- Scope: 每个 watcher 仍为非递归监听，但通过“展开目录时按需增加 watcher”支持懒加载递归浏览；未展开子树不扫描、不监听
