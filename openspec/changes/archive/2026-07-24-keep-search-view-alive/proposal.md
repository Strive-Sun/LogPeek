# Change: 优化搜索页面与启动生命周期

## Why

搜索页面目前在返回监控目录时被卸载，再次进入会重新读取搜索状态和配置，并短暂显示“搜索中”。同时，应用启动阶段会在核心后台状态发布前同步构造 `FileSearchManager`，打开搜索数据库和索引；这使非核心搜索模块与 UI、监控目录争夺首次启动资源。搜索页面应在进程内保活，搜索后端也应在核心功能就绪后按需后台初始化。

## What Changes

- 搜索页面首次打开后保留 React 组件实例，返回监控目录时仅隐藏，不卸载。
- 再次进入搜索页面时立即恢复原搜索词、结果、筛选、选中项、分页和滚动位置，不因页面切换重新查询。
- 隐藏页面不拦截监控目录的鼠标、键盘或可访问性焦点；再次显示时恢复搜索输入焦点。
- 保活范围仅限当前应用进程；退出应用后不把可能过期的搜索结果持久化到磁盘。
- 将轻量搜索启用偏好与重型搜索管理器拆分，应用优先启动 UI、监控目录和日志会话。
- 核心功能就绪后，仅在本次启动已启用搜索时通过后台阻塞任务初始化搜索数据库、索引和 provider；搜索关闭时完全跳过重型初始化。
- 搜索初始化不得阻塞 UI 响应或监控目录开始工作；初始化期间的搜索命令等待搜索后端就绪，不影响其他功能命令。

## Impact

- Affected specs: `file-search`, `application-lifecycle`
- Affected code: `src/App.tsx`、`src/components/FileSearchPanel.tsx`、Rust 应用状态与搜索初始化、前后端测试
