# 更新日志

本文档记录 LogPeek 每个正式版本的重要变更。发布说明按版本维护，并由 Release 工作流自动提取到 GitHub Releases。

## [Unreleased]

尚未发布的变更先记录在此；发布时移动到对应版本章节。

## [1.0.5] - 2026-07-18

### 新增

- 支持从系统文件管理器向主窗口拖入单个任意文件或文件夹并静默加入监控：文件使用父目录、文件夹使用自身作为监控根，不显示确认弹窗或全窗口拖入提示，原路径不会被复制或移动。
- 拖入 `.log`、`.txt`、`.json` 等可识别文本日志后立即打开正文；父目录已受监控或自动添加成功时，目录树会逐层展开并定位该文件。

### 优化

- 监控根按规范化路径和父子覆盖关系去重；新增父级根时自动归并已有子级根，避免同一目录树被重复监控。

### 修复

- 修复当前查看文件被后缀筛选隐藏时定位状态无法结束，导致重新展开监控根会误展开第一个子目录的问题。

## [1.0.4] - 2026-07-17

### 新增

- 点击主窗口关闭按钮时隐藏到系统托盘，目录监控和后台任务继续运行。
- 系统托盘提供“显示主窗口”和“退出 LogPeek”操作，只有显式退出才会结束应用进程。
- 监控根目录任意深度出现新的 zip 或匹配后缀日志文件时，即使对应子目录尚未展开也会触发新日志提示。
- 点击深层目录的新日志通知时，目录树会逐层展开目标路径、选中文件并滚动到可见区域。
- 深层新日志尚未加载到目录树时，折叠的监控根和已加载祖先仍会显示未读高亮。

### 优化

- 递归到达监听采用重叠根合并、后缀预筛、路径去重、固定 worker 和有界队列，避免空闲轮询及事件溢出时全盘重扫。

### 修复

- 将删除目录和删除文件的确认操作改为应用内弹窗，避免 WebView 原生确认窗口未显示时直接执行危险操作。
- 修复通知定位状态残留导致目录视觉上已展开、后台 watcher 却已释放，从而漏掉后续目录树实时更新的问题。

## [1.0.3] - 2026-07-17

### 新增

- 监控目录中的文件与文件夹在外部创建、删除、重命名或修改后，左侧目录树会实时同步；子目录支持展开时按需加载和监听。

### 修复

- 删除或重命名当前查看的文件时，自动清理失效通知和旧路径视图，避免目录树残留过期内容。

### 优化

- 目录事件按短时间窗口合并并仅更新受影响目录，稳定检测改为独立有界后台队列，避免批量文件操作触发重复全量扫描。

## [1.0.2] - 2026-07-17

### 新增

- 新增设置面板，展示当前应用版本并支持启动自动检查和手动检查更新。
- 发现新版本时支持跳过指定版本，或查看下载进度后自动验证、安装并重启应用。

### 工程质量

- Release 流程增加 Tauri updater 更新清单、双平台更新包和强制签名校验。

## [1.0.1] - 2026-07-17

### 优化

- 将应用版本号统一为 `src-tauri/Cargo.toml` 中的单一人工维护来源。
- GitHub Release 自动使用当前版本的逐条更新内容作为发行说明。

### 工程质量

- 新增发布日志校验，确保 Cargo 版本、Git tag 和 CHANGELOG 版本一致。
- 在 CI 中强制检查每个版本至少包含一条具体更新，拒绝模糊占位内容。
- 补充标准发布步骤，明确版本号、更新日志、检查、提交和 tag 的操作顺序。

## [1.0.0] - 2026-07-17

### 新增

- 支持同时监控多个本地目录，并在应用重启后恢复监控配置。
- 支持识别 zip 日志归档和裸文本日志，在目录树中按需展开归档条目。
- 支持无需手动解压直接查看归档内日志，并通过虚拟滚动浏览大文件。
- 支持后台增量建立行索引，实时展示进度并在索引过程中读取已就绪内容。
- 支持 UTF-8、GBK、GB18030、UTF-16LE 和 UTF-16BE 编码检测与手动切换。
- 支持 LF、CRLF、空行、超长行截断以及末行无换行符的日志内容。
- 支持日志文件和监控目录的重命名、定位、移除监控及删除到系统回收站。
- 支持后缀筛选、新日志通知、未读高亮、深浅主题和可拖拽目录栏。

### 优化

- zip Deflate 条目改为固定缓冲流式读取，避免大条目完整载入内存。
- zip Stored 条目支持受条目边界约束的字节 seek。
- 查看会话支持关闭、取消后台任务和 LRU 回收，自动清理临时缓存。
- 编码切换在后台重建行索引，并通过 generation 防止旧任务覆盖最新选择。

### 修复

- 修复打开失败后界面停留在“打开中”的问题。
- 修复重命名或删除文件后当前视图残留旧状态的问题。
- 修复前后端后缀筛选规则大小写及启动状态不一致的问题。
- 修复加密 zip 条目在目录枚举阶段无法作为元数据展示的问题。

### 工程质量

- 增加 20 个 Rust 单元测试，覆盖归档读取、目录监控、行索引、编码和资源清理。
- 建立 Rust fmt、Clippy、Cargo test、Prettier、ESLint 和前端构建的跨平台 CI 检查。
- 建立 Windows 与 macOS 安装包的 GitHub Release 自动构建流程。
