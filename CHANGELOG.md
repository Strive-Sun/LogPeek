# 更新日志

本文档记录 LogCrate 每个正式版本的重要变更。发布说明按版本维护，并由 Release 工作流自动提取到 GitHub Releases。

## [Unreleased]

尚未发布的变更先记录在此；发布时移动到对应版本章节。

## [1.0.14] - 2026-07-23

### 新增

- macOS 首次启动新增文件访问说明，可由用户主动打开“完全磁盘访问权限”系统设置，并可在应用设置中再次进入。
- macOS 监控目录新增 security-scoped bookmark 持久授权，支持跨重启恢复访问以及对单个失效目录重新授权。

### 优化

- 搜索、目录监控和日志打开共享持久授权状态；受保护目录不可读时提供明确修复入口，不再循环弹出授权窗口。

### 工程质量

- 新增 macOS 文件访问错误分类、旧监控配置迁移和首次权限引导测试，并保持 Windows 构建行为不变。
- 收紧 Windows NTFS 索引辅助代码的条件编译范围，修复 macOS 测试目标因未使用代码警告导致的 CI 失败。

## [1.0.13] - 2026-07-23

### 修复

- 将 NTFS 索引服务二进制限制为仅在 Windows Tauri 构建中启用，修复 macOS 通用安装包因尝试复制 Windows 专用服务而打包失败的问题。
- 清理搜索模块的平台条件编译边界，避免 macOS release 构建产生仅供 Windows 使用的未引用代码警告。

### 工程质量

- CI 新增 Windows 索引服务 feature 的独立编译检查，防止平台专用二进制从常规跨平台检查中遗漏。

## [1.0.12] - 2026-07-23

### 新增

- 新增全局本地文件搜索，可按文件名、路径、类型和扩展名筛选，并在索引仍在建立时持续展示来自多个磁盘卷的可用结果。
- 搜索结果支持双击打开日志或归档进行预览，并可通过右键菜单将文件所在目录加入 LogCrate 监控。
- Windows NTFS 固定卷新增基于 MFT 与 USN Journal 的快速索引服务，同时提供逐卷 provider 状态、降级原因和服务修复入口。

### 优化

- 搜索热路径迁移到 Orange 衍生的 Tantivy 架构，支持中文、拼音、多词 AND 查询、稳定相关性排序和后台增量提交。
- 全量重建采用原子索引快照切换，构建期间保留旧结果可查，并能在切换中断后恢复上一份完整索引。
- 可见搜索结果的大小、修改时间、可读性和内容类型由有限后台任务按需补齐，避免阻塞 UI 线程。

### 工程质量

- 为 Windows 索引服务增加受限 named-pipe IPC、并发与消息上限、断线取消、MFT/USN 边界测试及安全威胁模型。
- MSI 与 NSIS 安装包接入索引服务安装、修复和卸载配置，并随包分发 Orange GPLv3 与 UnRAR 许可证说明。
- 新增搜索面板交互测试，覆盖查询、双击打开、右键加入监控及失效结果处理。

## [1.0.11] - 2026-07-22

### 修复

- 修复 macOS 上 RAR 路径宽字符指针的符号位类型不匹配，导致通用 Apple 构建无法编译的问题。

## [1.0.10] - 2026-07-22

### 新增

- 启动时立即在原生主窗口显示动态进度条，WebView 完成加载后在同一窗口内无缝接管，消除冷启动期间的白屏。
- 持久化已打开选项卡、显示与折叠顺序及活动项，应用重启后仅立即加载活动日志，其余选项卡按需恢复。
- 已打开的裸日志或归档在外部修改时提示是否重新加载；外部删除时保留当前只读缓存，并可将当前日志内容另存为本地文件。

### 优化

- 将业务状态、缓存准备与目录 watcher 恢复移到后台执行，前端命令异步等待就绪，避免阻塞 UI 线程。
- 移除启动交接后会话恢复状态的大号刷新符号，仅在打开失败时保留警告提示。

### 修复

- 修复目录树中选中、定位与未读状态叠加时同时出现多行背景色的问题。
- 修复选项卡键盘焦点框只包裹标题文字、未覆盖完整标签的问题。

## [1.0.9] - 2026-07-20

### 修复

- 修复小日志索引完成事件被首次订阅消费后无法重放，导致 React StrictMode 或组件重新订阅时进度条长期停留在 0% 的问题；普通日志与压缩包条目现在使用准确的索引文案。

### 新增

- 新增 7z、单卷 RAR4/RAR5、TAR、tar.gz/bz2/xz/zst 与 gzip/bzip2/xz/zstd 单文件压缩流的只读查看支持。
- 支持跨格式嵌套归档惰性展开：只有用户主动展开当前节点时才流式读取下一层，并限制最大深度、累计解码字节、条目数量和危险路径。

### 文档

- 在中英文 README 增加自动适配 GitHub 深浅主题的主界面截图，展示监控目录、ZIP 展开、多选项卡和日志正文。
- 新建 `resources/icons/app` 与 `resources/screenshots` 分类管理图标和产品截图，并同步桌面构建与文档引用路径。
- 精简中英文 README 的项目介绍，聚焦当前产品名称与实际功能。
- 在长期路线图中新增 AI 辅助日志分析方向，并明确选区分析、模型配置与发送前脱敏边界。

### 测试

- 增加可注入的归档安全限制与自动化边界测试，覆盖嵌套深度和累计字节、扫描预算、危险路径、链接/设备节点及缓存清理。
- 增加可重复执行的 Windows release 性能基线，记录 256 MiB 裸日志、ZIP、tar.gz、tar.zst 和跨格式嵌套归档的首批行延迟、吞吐、CPU、内存及磁盘缓存。

## [1.0.7] - 2026-07-20

## [1.0.6] - 2026-07-20

### 新增

- 打开多个裸日志或压缩包日志条目时创建独立选项卡，并分别保留正文滚动位置、索引进度、编码和有限行缓存。
- 正文宽度不足时将溢出选项卡收入带数量提示的“更多”折叠栏；选择折叠项会替换最右侧可见选项卡，并保持稳定顺序。

### 优化

- 鼠标悬停在正文顶部已打开文件名时显示磁盘绝对路径；压缩包日志同时显示压缩包绝对路径和包内路径。
- 后端继续限制最多 5 个活动日志会话；超出上限的选项卡进入休眠，重新激活时透明恢复会话与用户编码选择，避免资源随选项卡数量无限增长。
- 目录树、日志正文和可滚动菜单默认隐藏滚动条，滚动时立即显示，并在停止操作后延迟渐隐。

## [1.0.5] - 2026-07-18

### 新增

- 支持从系统文件管理器向主窗口拖入单个任意文件或文件夹并静默加入监控：文件使用父目录、文件夹使用自身作为监控根，不显示确认弹窗或全窗口拖入提示，原路径不会被复制或移动。
- 拖入 `.log`、`.txt`、`.json` 等可识别文本日志后立即打开正文；父目录已受监控或自动添加成功时，目录树会逐层展开并定位该文件。

### 优化

- 监控根按规范化路径和父子覆盖关系去重；新增父级根时自动归并已有子级根，避免同一目录树被重复监控。

### 修复

- 修复当前查看文件被后缀筛选隐藏时定位状态无法结束，导致重新展开监控根会误展开第一个子目录的问题。

## [1.0.4] - 2026-07-17

### 新增

- 点击主窗口关闭按钮时隐藏到系统托盘，目录监控和后台任务继续运行。
- 系统托盘提供“显示主窗口”和“退出 LogCrate”操作，只有显式退出才会结束应用进程。
- 监控根目录任意深度出现新的 zip 或匹配后缀日志文件时，即使对应子目录尚未展开也会触发新日志提示。
- 点击深层目录的新日志通知时，目录树会逐层展开目标路径、选中文件并滚动到可见区域。
- 深层新日志尚未加载到目录树时，折叠的监控根和已加载祖先仍会显示未读高亮。

### 优化

- 递归到达监听采用重叠根合并、后缀预筛、路径去重、固定 worker 和有界队列，避免空闲轮询及事件溢出时全盘重扫。

### 修复

- 将删除目录和删除文件的确认操作改为应用内弹窗，避免 WebView 原生确认窗口未显示时直接执行危险操作。
- 修复通知定位状态残留导致目录视觉上已展开、后台 watcher 却已释放，从而漏掉后续目录树实时更新的问题。

## [1.0.3] - 2026-07-17

### 修复

### 优化

- 目录事件按短时间窗口合并并仅更新受影响目录，稳定检测改为独立有界后台队列，避免批量文件操作触发重复全量扫描。

## [1.0.0] - 2026-07-17

### 新增

- 支持同时监控多个本地目录，并在应用重启后恢复监控配置。
- 支持识别 zip 日志归档和裸文本日志，在目录树中按需展开归档条目。
- 支持无需手动解压直接查看归档内日志，并通过虚拟滚动浏览大文件。
- 支持后台增量建立行索引，实时展示进度并在索引过程中读取已就绪内容。
- 支持 UTF-8、GBK、GB18030、UTF-16LE 和 UTF-16BE 编码检测与手动切换。
- 支持 LF、CRLF、空行、超长行截断以及末行无换行符的日志内容。
- 支持日志文件和监控目录的重命名、定位、移除监控及删除到系统回收站。
- 支持后缀筛选、新日志通知、未读高亮、深浅主题和可拖拽目录栏。

### 优化

- zip Deflate 条目改为固定缓冲流式读取，避免大条目完整载入内存。
- zip Stored 条目支持受条目边界约束的字节 seek。
- 查看会话支持关闭、取消后台任务和 LRU 回收，自动清理临时缓存。
- 编码切换在后台重建行索引，并通过 generation 防止旧任务覆盖最新选择。

### 修复

- 修复打开失败后界面停留在“打开中”的问题。
- 修复重命名或删除文件后当前视图残留旧状态的问题。
- 修复前后端后缀筛选规则大小写及启动状态不一致的问题。
- 修复加密 zip 条目在目录枚举阶段无法作为元数据展示的问题。

### 工程质量

- 增加 20 个 Rust 单元测试，覆盖归档读取、目录监控、行索引、编码和资源清理。
- 建立 Rust fmt、Clippy、Cargo test、Prettier、ESLint 和前端构建的跨平台 CI 检查。
- 建立 Windows 与 macOS 安装包的 GitHub Release 自动构建流程。
